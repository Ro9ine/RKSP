# Анализ предметной области

## Тема

Клиент-серверное приложение управления рабочими процессами (workflow management) в рамках деятельности IT-команды.

## Проблема

Руководителю команды (тимлиду) необходимо распределять задачи между сотрудниками, задавать сроки выполнения и получать обратную связь о результате. Сотрудникам нужен единый интерфейс для просмотра назначенных задач, фиксации прогресса и отчёта о выполнении.

## Участники и роли

| Роль | Описание |
|------|----------|
| Teamlead | Создаёт учётные записи сотрудников, формирует команду, назначает задачи, задаёт срок (от 1 часа до 14 суток), просматривает статусы и комментарии |
| Employee | Просматривает свои задачи, переводит в «в работе», добавляет комментарии, завершает задачу с отметкой успех/неуспех и обязательным комментарием |

## Основные сущности

- **Пользователь** — учётная запись с ролью и аутентификацией.
- **Команда** — группа сотрудников под управлением одного тимлида.
- **Задача** — единица работы с названием, описанием, исполнителем, сроком и статусом.
- **Комментарий** — текстовое сообщение сотрудника по задаче.
- **Результат выполнения** — итог (успех/неуспех) с пояснением при закрытии задачи.

## Сценарии использования

1. **Авторизация** — пользователь входит по email и паролю, получает JWT.
2. **Создание сотрудника** — тимлид регистрирует сотрудника и автоматически добавляет его в команду.
3. **Назначение задачи** — тимлид выбирает исполнителя из команды и указывает срок в часах или днях.
4. **Выполнение задачи** — сотрудник берёт задачу в работу, комментирует, завершает с результатом.
5. **Контроль** — тимлид видит список задач команды и итоги.

## Ограничения и правила

- Срок задачи: от 60 минут до 14 суток (20160 минут).
- Сотрудник не может назначать задачи и управлять чужой командой.
- Завершённую задачу нельзя редактировать.
- При завершении обязателен непустой комментарий.

## Выбранная архитектура

Трёхзвенная клиент-серверная архитектура:

- **Клиент** — SPA (React), взаимодействует с API по HTTPS/JSON.
- **Сервер приложений** — FastAPI, бизнес-логика, JWT, RBAC.
- **СУБД** — PostgreSQL, хранение данных.

Преимущества: разделение ответственности, независимое развитие клиента и сервера, масштабируемость API, автогенерация OpenAPI для тестирования.
