# Презентация к защите (структура слайдов)

1. **Титул** — тема, ФИО, группа, руководитель.
2. **Цель и задачи** — автоматизация управления задачами команды.
3. **Анализ предметной области** — роли, сущности (диаграмма из `docs/uml/use-case.puml`).
4. **Архитектура** — клиент-сервер, component diagram.
5. **Стек технологий** — React, FastAPI, PostgreSQL, Docker.
6. **UML** — sequence: login, complete task; class diagram.
7. **Демонстрация** — скриншоты: вход, панель тимлида, панель сотрудника.
8. **Безопасность** — JWT, RBAC, валидация сроков.
9. **Fuzz-тестирование** — schemathesis, основные результаты (`docs/fuzz-report.md`).
10. **Развёртывание** — Docker Compose локально; планы облака (Render/VPS) — следующий этап.
11. **Выводы** — достигнутые результаты, перспективы.

## Скриншоты для вставки

- `/login` — форма входа
- `/teamlead` — команда + создание задачи
- `/employee` — список задач + завершение
- `/docs` — Swagger UI
